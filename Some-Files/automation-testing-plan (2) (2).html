🚀 خطوات الاشتغال في الأتمتة

roadmap.md
PHASE 01
الإعداد
  • اختاروا أداة واحدة للفريق كله (Selenium أو Playwright)
  • مشروع واحد على GitHub، كل حد على branch خاص بمجموعته
  • ملف config موحّد للبيئات (URL, credentials, browser)
  • هيكل مجلدات موحّد قبل ما حد يكتب أول سطر كود
PHASE 02
التصميم
  • Page Object Model — كل صفحة/موديول في كلاس منفصل
  • تسمية موحّدة للـ tests والـ methods (نفس الـ convention للكل)
  • Locators في مكان مركزي، مش متكررة جوه كل test
  • كل مجموعة تعمل الـ Page Objects بتاعتها في مجلدها فقط
PHASE 03
البناء
  • اكتبوا الـ helpers المشتركة أول حاجة (login helper, setup/teardown)
  • فصلوا بيانات الاختبار عن الكود (JSON / Excel / .env)
  • استخدموا Explicit Waits، تجنبوا Thread.sleep نهائيًا
  • كل test case يبقى مستقل، متعتمدش على test قبله
PHASE 04
التنفيذ
  • Tag كل test بـ smoke / regression عشان تشغلوا اللي محتاجينه بس
  • شغّلوا الاختبارات parallel لو الأداة بتدعم كده
  • اربطوا المشروع بـ CI/CD (GitHub Actions) لتشغيل تلقائي
  • مراجعة يومية سريعة (15 دقيقة) لمتابعة الـ blockers
PHASE 05
التقارير
  • استخدموا أداة تقارير زي Allure أو Extent Reports
  • Screenshot تلقائي عند فشل أي test
  • Pull Request + code review قبل الدمج على main
  • وثّقوا كل bug اتكشف أثناء الأتمتة في تراكر واحد
